sintra::shutdown

Include:

#include <sintra/sintra.h>

Summary:

shutdown is the standard terminal API for the multi-process Sintra lifecycle. All live participants enter the same collective shutdown protocol; the runtime owns the rendezvous, optional coordinator hook, drain wait, and teardown. The simple form is a symmetric collective shutdown. The overload that takes shutdown_options lets the coordinator run a bounded side effect inside the protocol.

Signature:

bool shutdown();
bool shutdown(const shutdown_options& options);

Use when:

  • All swarm participants reach the same top-level handoff and are ready to tear down together.
  • The coordinator must run a bounded side effect (write a summary file, flush metrics) at a defined point inside the shutdown protocol; pass a shutdown_options with a coordinator_shutdown_hook.
  • A single-process program needs the public terminal API. With no coordinator id present, shutdown() falls through to detail::finalize internally without a collective barrier.

Contract:

  • shutdown() is equivalent to shutdown(shutdown_options{}).
  • The collective protocol runs three steps:
  • A processing-fence barrier on the _sintra_all_processes group.
  • The coordinator runs coordinator_shutdown_hook if set, while non-coordinator participants wait inside the protocol.
  • The coordinator drain-waits for peers to exit the group, then every participant tears down its local managed process.
  • Returns true when teardown completed locally. Returns false when no managed process was active to tear down.
  • A second shutdown() call while a teardown is already in progress is rejected: the runtime throws std::logic_error to flag the illegal composition.
  • Ordinary callers must not pair shutdown() with extra final _sintra_all_processes user barriers, additional detail::finalize calls, or duplicated shutdown invocations.
  • After successful return, the local Sintra runtime is torn down: facade APIs are no longer usable until a new init is performed.

Threading and lifecycle:

  • Call from a top-level control thread, never from inside a message handler or post-handler callback. The protocol claims teardown admission state synchronously.
  • The collective barrier blocks until every live participant arrives. Participants that have already left via leave() or have exited abnormally are excluded from the membership snapshot.
  • If the coordinator hook throws, the runtime still finalises the local process and rethrows the exception to the caller of shutdown(options).

Failures:

  • Throws std::logic_error when called while another lifecycle teardown is already in progress.
  • Propagates exceptions thrown by coordinator_shutdown_hook to the coordinator's caller after finalising local state.
  • A barrier RPC failure inside the protocol is treated as satisfied during shutdown handling and does not abort teardown.
